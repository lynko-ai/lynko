# Contributing to Lynko

Thanks for your interest in Lynko! This guide explains how to contribute.

## Reporting Bugs

Use the [bug report template](../../issues/new?template=bug_report.yml) to file a bug. Good bug reports include:

- The exact DSL command you ran
- What happened vs. what you expected
- Your MCP client and collection type
- Whether the issue reproduces consistently

Please search [existing issues](../../issues) before filing a new one.

## Requesting Features

Use the [feature request template](../../issues/new?template=feature_request.yml). Describe the problem you're trying to solve, not just the solution you want. This helps us understand the use case and find the best approach.

## Security Vulnerabilities

Do **not** file a public issue for security vulnerabilities. See [SECURITY.md](SECURITY.md) for responsible disclosure instructions.

## Discussions

For questions, ideas, and general conversation that aren't actionable bugs or feature requests, use [GitHub Discussions](../../discussions). The Q&A category is a good place to ask "how do I..." questions.

## Skills and Documentation

Found an error in the docs or skills? File an issue or open a pull request. We welcome improvements to:

- Onboarding guides
- DSL cheatsheet and FAQ
- Agent skill instructions
- Typos and clarity fixes

## Code of Conduct

Be respectful. We're a small team building something new, and we value constructive feedback. Harassment, personal attacks, and bad-faith engagement will not be tolerated.
